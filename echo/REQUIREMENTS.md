# echo 요구사항

## 목적

이 프로젝트는 Zig 0.16.0으로 Unix 계열 CLI 도구를 학습하기 위한 `echo` 구현이다.
목표는 단순히 문자열을 출력하는 프로그램을 만드는 것이 아니라, CLI 인자 처리, 표준 출력, 종료 코드, 테스트 가능한 동작 정의를 익히는 것이다.

## 범위

### 기본 범위

- 실행 파일 이름은 `echo`이다.
- 명령행 인자로 받은 문자열들을 표준 출력(stdout)에 출력한다.
- 인자 사이에는 공백 문자 하나(` `)를 출력한다.
- 기본적으로 출력 끝에 개행 문자(`\n`)를 출력한다.
- 표준 입력(stdin)은 읽지 않는다.
- 정상 실행 시 종료 코드는 `0`이다.
- 출력 실패 등 I/O 오류가 발생하면 비정상 종료한다.

### 학습용 확장 범위

- `-n` 옵션을 지원한다.
- `--help` 옵션을 지원한다.
- `--version` 옵션을 지원한다.
- 이스케이프 해석(`-e`, `-E`)은 기본 구현 이후 선택 과제로 둔다.

## 명령 형식

```text
echo [OPTION]... [STRING]...
```

## 기능 요구사항

### 1. 인자 출력

- 인자가 없으면 개행 문자만 출력한다.
- 인자가 하나면 해당 인자를 그대로 출력한 뒤 개행한다.
- 인자가 둘 이상이면 각 인자를 공백 하나로 구분해 출력한 뒤 개행한다.
- 쉘이 넘겨준 인자를 그대로 사용한다.
- 따옴표 처리, 변수 치환, 글로브 확장 등은 쉘의 책임이며 `echo`가 직접 처리하지 않는다.

예시:

```sh
$ echo

$ echo hello
hello

$ echo hello world
hello world
```

### 2. `-n` 옵션

- 첫 번째 비프로그램 인자가 정확히 `-n`이면 마지막 개행을 출력하지 않는다.
- `-n` 뒤의 나머지 인자들은 일반 문자열로 출력한다.
- 기본 학습 범위에서는 여러 개의 `-n`을 모두 옵션으로 해석하지 않아도 된다.
- `-n`이 첫 번째 인자가 아니면 일반 문자열로 출력한다.

예시:

```sh
$ echo -n hello
hello

$ echo hello -n
hello -n
```

### 3. `--help` 옵션

- 첫 번째 비프로그램 인자가 정확히 `--help`이면 사용법을 stdout에 출력하고 종료 코드 `0`으로 종료한다.
- `--help` 뒤에 다른 인자가 있어도 도움말 출력이 우선한다.
- 도움말에는 최소한 사용 형식, 지원 옵션, 간단한 설명이 포함되어야 한다.

예시 출력 형식:

```text
Usage: echo [OPTION]... [STRING]...
Print STRING(s) to standard output.

Options:
  -n         do not output the trailing newline
  --help     display this help and exit
  --version  output version information and exit
```

### 4. `--version` 옵션

- 첫 번째 비프로그램 인자가 정확히 `--version`이면 버전 정보를 stdout에 출력하고 종료 코드 `0`으로 종료한다.
- `--version` 뒤에 다른 인자가 있어도 버전 출력이 우선한다.
- 버전 문자열은 프로젝트 버전과 맞춘다.

예시:

```text
echo 0.0.0
```

### 5. 옵션으로 보이지 않는 문자열

- 기본 구현에서는 `--`를 특별 처리하지 않는다.
- `--`는 일반 문자열로 출력한다.
- 알 수 없는 옵션처럼 보이는 값도 일반 문자열로 출력한다.

예시:

```sh
$ echo -- hello
-- hello

$ echo -x hello
-x hello
```

## 선택 과제: 이스케이프 해석

기본 요구사항을 만족한 뒤 다음 옵션을 추가할 수 있다.

### `-e`

- 백슬래시 이스케이프 시퀀스를 해석한다.
- 최소 지원 후보:
  - `\\`: 백슬래시
  - `\n`: 개행
  - `\t`: 탭
  - `\r`: 캐리지 리턴
  - `\b`: 백스페이스
  - `\c`: 이후 출력을 중단하고 마지막 개행도 출력하지 않음

### `-E`

- 백슬래시 이스케이프 시퀀스를 해석하지 않는다.
- 기본 동작과 같다.

## 비기능 요구사항

### Zig 0.16.0 기준

- `build.zig.zon`의 `minimum_zig_version`은 `0.16.0`을 유지한다.
- 표준 라이브러리만 사용한다.
- stdout 출력은 Zig 0.16.0의 I/O API를 사용한다.
- 핵심 출력 로직은 테스트하기 쉽도록 함수로 분리하는 것을 권장한다.

### 안정성

- 빈 문자열 인자를 보존한다.
- 인자 내부의 공백, 탭, 유니코드 문자는 변경하지 않는다.
- 매우 긴 입력도 고정 크기 버퍼에 전부 복사하지 않고 스트리밍 방식으로 출력하는 것을 권장한다.
- stdout flush 실패는 오류로 취급한다.

### 호환성

- Linux, macOS 등 Unix 계열 환경에서 동작하는 것을 목표로 한다.
- Windows 호환성은 필수 범위가 아니다.
- POSIX `echo`는 구현별 차이가 있으므로, 이 프로젝트의 동작은 본 문서를 기준으로 고정한다.

## 테스트 요구사항

다음 동작은 자동 테스트 또는 수동 실행으로 확인해야 한다.

| 명령 | 기대 출력 | 비고 |
| --- | --- | --- |
| `echo` | `\n` | 인자 없음 |
| `echo hello` | `hello\n` | 인자 하나 |
| `echo hello world` | `hello world\n` | 인자 여러 개 |
| `echo ""` | `\n` | 빈 문자열 인자 |
| `echo hello "" world` | `hello  world\n` | 빈 인자 보존 |
| `echo -n hello` | `hello` | 개행 없음 |
| `echo hello -n` | `hello -n\n` | 중간 `-n`은 문자열 |
| `echo -- hello` | `-- hello\n` | `--`는 문자열 |
| `echo -x hello` | `-x hello\n` | 알 수 없는 옵션은 문자열 |
| `echo --help` | 도움말 출력 | 종료 코드 `0` |
| `echo --version` | 버전 출력 | 종료 코드 `0` |

## 완료 기준

- `zig build`가 성공한다.
- `zig build run -- hello world`가 `hello world`와 개행을 출력한다.
- `zig build test`가 성공한다.
- 문서의 테스트 요구사항을 만족한다.
- 오류 발생 시 Zig 오류 전파 또는 명확한 비정상 종료가 이루어진다.
